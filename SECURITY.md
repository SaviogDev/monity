# 🛡️ Política de Segurança do Monity

A segurança dos dados financeiros dos nossos usuários é a prioridade número um do **Monity**. Nós levamos todas as vulnerabilidades e falhas de segurança muito a sério.

## 🟢 Versões Suportadas

Atualmente, garantimos atualizações de segurança e correções de bugs para as seguintes versões da nossa plataforma:

| Versão | Suportada | Notas |
| :--- | :--- | :--- |
| >= 1.0.x (Beta) | ✅ Sim | Versão atual em produção |
| < 1.0.0 | ❌ Não | Versões de teste alfa |

## 🚨 Como reportar uma vulnerabilidade

**Por favor, NÃO abra uma "Issue" pública no GitHub para reportar falhas de segurança.**

Se você encontrou uma vulnerabilidade, um bug crítico ou uma brecha de segurança, pedimos que reporte diretamente para a nossa equipe de desenvolvimento de forma privada. 

1. Envie um e-mail detalhado para: **[saviog.dev@gmail.com]**
2. Coloque no assunto: `[ALERTA DE SEGURANÇA] - Monity`
3. No corpo do e-mail, inclua:
   - Um resumo do problema.
   - Os passos exatos para reproduzir a falha.
   - O impacto potencial da vulnerabilidade.

### Nosso compromisso:
- Confirmaremos o recebimento do seu e-mail em até **48 horas**.
- Enviaremos atualizações sobre o progresso da correção.
- Daremos os devidos créditos a você (se desejar) quando a correção for publicada.

Agradecemos por ajudar a manter o ecossistema Monity seguro para todos! 🥂
